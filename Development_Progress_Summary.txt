+ 一、项目概述
+ ================================================================================
+ 
+ 1.1 项目目标
+ --------------------------------------------------------------------------------
+ 基于MobileNetV2架构，实现Block2~6全光计算的MNIST分类任务：
+ - 教师模型：标准MobileNetV2（全电子计算，float32精度）
+ - 学生模型：Block2~6全光计算（Pace2的8x8/32x32混合模式）
+ - 知识蒸馏：通过教师模型的软标签补偿量化精度损失
+ 
+ 1.2 目标指标
+ --------------------------------------------------------------------------------
+ - 教师模型准确率：>97%（实际达到99.47%）
+ - 学生模型准确率：>85%
+ - 光计算占比：>80%
+ 
+ 
+ 二、开发环境
+ ================================================================================
+ 
+ 2.1 硬件配置
+ --------------------------------------------------------------------------------
+ - CPU：AMD Ryzen 9 8940HX，16核32线程
+ - GPU：NVIDIA GeForce RTX 5060 Laptop（8GB显存）
+ - 内存：16GB
+ - 操作系统：WSL2 + Docker Desktop + Ubuntu 22.04.5
+ 
+ 2.2 软件环境
+ --------------------------------------------------------------------------------
+ - Python：3.9.5
+ - PyTorch：2.1.0+cu121
+ - 虚拟环境：conda环境 moca_llm
+ - 光计算模拟器：LTSimulator（暂无授权码）
+ 
+ 2.3 重要说明
+ --------------------------------------------------------------------------------
+ 【CUDA兼容性问题】
+ RTX 5060是Blackwell架构（sm_120），PyTorch 2.1.0仅支持到sm_90（Hopper）。
+ 当前解决方案：强制使用CPU训练（force_cpu=True）
+ 
+ 【后续解决方案】
+ 需要升级PyTorch到支持Blackwell架构的版本（PyTorch 2.5+）
+ 
+ 
+ 三、技术方案要点
+ ================================================================================
+ 
+ 3.1 模型架构
+ --------------------------------------------------------------------------------
+ Block    类型           计算方式      光计算模型    输入通道    输出通道
+ --------------------------------------------------------------------------------
+ Block1   初始卷积       电子计算      -            1           32
+ Block2   Inverted x1    全光计算      Pace2(8x8)   32          16
+ Block3   Inverted x2    全光计算      Pace2(8x8/32x32)  16      24
+ Block4   Inverted x3    全光计算      Pace2(32x32) 24          32
+ Block5   Inverted x4    全光计算      Pace2(32x32) 32          64
+ Block6   Inverted x3    全光计算      Pace2(32x32) 64          96
+ Block7   输出卷积+分类器 电子计算      -            96          10
+ --------------------------------------------------------------------------------
+ 
+ 3.2 量化策略
+ --------------------------------------------------------------------------------
+ 量化对象      量化类型    量化范围      量化方式
+ --------------------------------------------------------------------------------
+ 输入激活      uint4      [0, 15]      Min-Max动态量化
+ 权重          int4       [-8, 7]      对称量化
+ 输出          float32    -            反量化
+ --------------------------------------------------------------------------------
+ 
+ 3.3 知识蒸馏参数
+ --------------------------------------------------------------------------------
+ - 软标签权重 α：0.7
+ - 温度参数 T：3.0
+ - 损失函数：L_total = α × L_soft + (1-α) × L_hard
+ - 学习率：0.0005
+ - 训练轮数：30 epochs（建议可减少到15-20）

+ 六、osimulator API使用要点
+ ================================================================================
+ 
+ 6.1 模型实例路径
+ --------------------------------------------------------------------------------
+ osimulator/models/pace2/
+ ├── 8X8_4a4w8o_dacenob5.5_power0.01_noise2e-11_1gclock_mean-1.11_std17
+ └── 32X2_4a4w8o_dacenob5.5_power0.02_noise2e-11_1gclock_mean-5.95_std38.3
+ 
+ 6.2 加载模型示例
+ --------------------------------------------------------------------------------
+ import os
+ import torch
+ import osimulator
+ from osimulator.api import load_approach_model
+ 
+ # 获取osimulator路径
+ module_path = os.path.abspath(osimulator.__file__)
+ osim_path = os.path.dirname(module_path)
+ 
+ # 加载8x8模型
+ pace2_8x8_path = os.path.join(osim_path, "models/pace2/8X8_4a4w8o_...")
+ model_8x8 = load_approach_model(device_type="cpu", instances_path=pace2_8x8_path)
+ model_8x8.int2uint = True  # int4输入时必须设置
+ 
+ # 加载32x32模型
+ pace2_32x32_path = os.path.join(osim_path, "models/pace2/32X2_4a4w8o_...")
+ model_32x32 = load_approach_model(device_type="cpu", instances_path=pace2_32x32_path)
+ model_32x32.int2uint = True
+ 
+ 6.3 矩阵乘法API
+ --------------------------------------------------------------------------------
+ from osimulator.api import pace2_8x8and16x16mode_matmul, pace2_32x32mode_matmul
+ 
+ # 8x8模式
+ result = pace2_8x8and16x16mode_matmul(
+     x=x,                      # torch.int32, uint4范围[0,15]
+     y=y,                      # torch.int32, int4范围[-8,7]
+     pace2_instance=model_8x8,
+     inputType="uint4"
+ )
+ 
+ # 32x32模式
+ result = pace2_32x32mode_matmul(
+     x=x, y=y,
+     pace2_instance=model_32x32,
+     inputType="uint4"
+ )
+ 
+ 6.4 动态选择规则
+ --------------------------------------------------------------------------------
+ - 通道数 ≤ 16：使用8x8模式
+ - 通道数 > 16：使用32x32模式
